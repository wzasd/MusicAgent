## 2. 数据流架构图

```
用户请求处理流程:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

用户输入: "我想听适合运动时听的歌曲"
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│  1. 前端层 (Next.js)                                                            │
│     • 用户在输入框输入请求                                                       │
│     • ChatInput 组件捕获输入                                                     │
│     • lib/api.ts 的 streamRecommendations 发起 SSE 连接                          │
│     • 通过 /api/recommendations/stream 代理到后端                                  │
└─────────────────────────────────────────────────────────────────────────────────┘
    │
    ▼ SSE (EventStream)
┌─────────────────────────────────────────────────────────────────────────────────┐
│  2. 后端 API 层 (FastAPI)                                                        │
│     • api/server.py 接收 POST /api/recommendations/stream                         │
│     • stream_recommendations() 异步生成器函数                                     │
│     • 创建 StreamingResponse 返回 SSE 流                                          │
│     • 逐事件发送: start → thinking → response → song → complete                   │
└─────────────────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│  3. Agent 层 (MusicRecommendationAgent)                                          │
│     • get_recommendations(query) 被调用                                          │
│     • 构建 MusicAgentState 初始状态                                               │
│     • 调用 LangGraph 工作流: app.ainvoke(state)                                   │
└─────────────────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│  4. LangGraph 工作流层                                                           │
│                                                                                  │
│   ┌──────────────┐                                                              │
│   │ analyze_intent │  LLM分析意图                                                │
│   │  意图类型:    │  → "recommend_by_activity"                                    │
│   │ recommend_by_ │  → 参数: {"activity": "运动", "mood": "energetic"}              │
│   │   activity    │                                                              │
│   └──────┬───────┘                                                              │
│          │                                                                       │
│          ▼ route_by_intent                                                       │
│   ┌──────────────┐                                                              │
│   │generate_recom-│  调用 MusicSearchTool                                        │
│   │mendations    │  • Spotify: get_recommendations(seed_genres=['workout'])       │
│   │              │  • 或 TailyAPI: 搜索 "运动音乐 节奏感强"                        │
│   └──────┬───────┘                                                              │
│          │                                                                       │
│          ▼                                                                       │
│   ┌──────────────┐                                                              │
│   │generate_expla-│  LLM生成推荐解释                                             │
│   │  nation      │  "根据你的运动场景，为你推荐以下节奏强劲的歌曲..."               │
│   └──────────────┘                                                              │
│          │                                                                       │
│          ▼                                                                       │
│   返回 MusicAgentState (包含 recommendations, explanation, final_response)        │
└─────────────────────────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────────────────────────┐
│  5. 响应流式输出                                                                  │
│     • 后端将结果拆分为多个 SSE 事件                                               │
│     • 前端逐事件接收并更新UI                                                      │
│     • ThinkingIndicator 显示处理状态                                              │
│     • ResultsDisplay 逐步展示歌曲                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

## 6. 关键设计模式

### 6.1 状态机模式 (LangGraph)
- 使用 `MusicAgentState` 作为工作流的共享状态
- 节点间通过状态传递数据
- 支持条件路由 (`route_by_intent`)

### 6.2 流式响应 (SSE)
- 后端使用 `StreamingResponse` 生成 SSE 流
- 前端使用 `EventSource` 接收流式事件
- 支持实时显示思考过程和推荐结果

### 6.3 多数据源适配
- 优先使用 Spotify API (专业音乐数据)
- 降级到 TailyAPI/Tavily (网络搜索)
- 最后使用本地 JSON 数据库

### 6.4 单例模式
- `MusicRecommendationAgent` 单例
- `PlaylistRecommendationService` 单例
- `MusicJourneyService` 单例
